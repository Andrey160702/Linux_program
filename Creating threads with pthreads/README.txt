Заходим на полигон Linux в каталог pthreads progs
1)Программа, создающая несколько потоков и выводящая на экран строку 10 раз.
Скомпилировать программу с помощью команды gcc 1.c -lpthread -o 1
Запустить программу с помощью команды ./1 n, где n - колличество потоков
2)Программа, создающая несколько потоков, каждый из которых в цикле увеличивает на один значение глобальной переменной программы без синхронизации, из-за чего результат будет неверным. (Каждый поток имеет 1 000 000 операция сложения).
Скомпилировать программу с помощью команды gcc 2.c -lpthread -o 2
Запустить программу с помощью команды ./2 n, где n - колличество потоков
3)Программа, создающая несколько потоков, каждый из которых в цикле увеличивает на один значение глобальной переменной программы c синхронизацией общей переменной с помощью мьютекса, из-за чего результат будет верным. (Каждый поток имеет 1 000 000 операция сложения).
Скомпилировать программу с помощью команды gcc 3.c -lpthread -o 3
Запустить программу с помощью команды ./3 n, где n - колличество потоков 
4)Oптимизация с помощью временной переменной программы, создающей несколько потоков, где каждый поток в цикле увеличивает на один значение глобальной переменной программы c синхронизацией общей переменной с помощью мьютекса, из-за чего результат будет верным и быстрым. (Каждый поток имеет 1 000 000 операция сложения).
Скомпилировать программу с помощью команды gcc 4.c -lpthread -o 4
Запустить программу с помощью команды ./4 n, где n - колличество потоков
